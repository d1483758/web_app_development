# 產品需求文件 (PRD) - 食譜收藏夾

## 1. 專案概述
- **背景與動機**：現代人生活忙碌，常常不知道該煮什麼，或是看到不錯的食譜卻找不到合適的地方統整儲存。本專案旨在提供一個方便的線上食譜平台，讓使用者能輕鬆尋找靈感並收藏喜愛的食譜。
- **目標用戶**：
  - **一般人（一般使用者）**：想要尋找食譜、記錄烹飪材料與步驟，並收藏喜歡食譜的人。
  - **管理員**：負責審核推薦內容、管理不當內容與維護平台秩序的人員。
- **核心價值主張**：提供直覺的食譜搜尋與收藏體驗，以及清晰詳盡的材料與步驟呈現，讓下廚變得更輕鬆簡單。

## 2. 功能需求

### 主要功能清單
1. 新增與編輯食譜
2. 瀏覽與搜尋食譜
3. 列出與檢視食譜的材料和步驟
4. 使用者註冊與登入
5. 個人收藏夾管理
6. 管理後台與內容管理

### 使用者故事 (User Stories)
- **功能 1：新增與編輯食譜**
  - 作為 **一般使用者**，我希望 **可以發布與編輯自己的食譜（包含名稱、材料、步驟）**，以便 **記錄自己的料理並分享給所有人**。
- **功能 2：瀏覽與搜尋食譜**
  - 作為 **一般使用者**，我希望 **可以透過關鍵字搜尋或列表瀏覽綜合食譜**，以便 **快速找到我想做的料理靈感**。
- **功能 3：列出與檢視食譜的材料和步驟**
  - 作為 **一般使用者**，我希望 **點開食譜後能看到清晰的材料清單與依序排列的步驟**，以便 **跟著指示順利完成料理購買與製作**。
- **功能 4：使用者註冊與登入**
  - 作為 **訪客**，我希望 **可以註冊並登入系統**，以便 **擁有個人身分來收藏食譜或發布內容**。
- **功能 5：個人收藏夾管理**
  - 作為 **一般使用者**，我希望 **能在看到喜歡的食譜時點擊收藏，並在個人頁面查看收藏夾**，以便 **未來需要料理時能快速查閱**。
- **功能 6：管理後台與內容管理**
  - 作為 **管理員**，我希望 **可以具備較高的權限來查看所有食譜與使用者列表，並能刪除違規內容**，以便 **維護整體站點的品質與秩序**。

## 3. 非功能需求
- **技術限制**：
  - 後端將使用 **Flask** 輕量級框架。
  - 前端將採用 **Jinja2** 作為模板引擎，搭配基礎 HTML/CSS/JS。
  - 資料庫使用 **SQLite**，適合輕量級應用與快速開發。
- **效能考量**：
  - 食譜列表應有基本的分頁或延遲載入機制，避免一次性查詢過大影響首頁載入速度。
  - 搜尋結果應於可接受範圍內（如 1 秒內）回傳。
- **安全考量**：
  - 使用者密碼需經過雜湊 (Hash) 處理後才存入資料庫（如 Werkzeug security）。
  - 對任何使用者輸入皆需進行驗證與過濾，防範基本的 SQL Injection 與 XSS 攻擊。

## 4. MVP 範圍 (Minimum Viable Product)

| 功能層級 | 功能描述 |
| --- | --- |
| **Must Have (必須有)** | - 使用者註冊/登入功能<br>- 瀏覽最新的食譜列表<br>- 檢視單個食譜的詳細材料與步驟<br>- 登入後可新增簡單食譜功能<br>- 基本的關鍵字搜尋 |
| **Should Have (應該有)** | - 將食譜加入/移除個人收藏夾功能<br>- 編輯或刪除自己發布的食譜<br>- 食譜分類（如：中式、西式、甜點等） |
| **Nice to Have (加分項)** | - 管理員專屬後台介面<br>- 食譜圖片上傳功能<br>- 留言板或評分機制<br>- 列印食譜或材料採購清單功能 |
